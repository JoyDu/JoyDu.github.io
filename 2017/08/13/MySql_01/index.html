MySql

MySql

影响数据库因素：
Sql查询速度(一条sql只能用到一个cpu)避免SELECT 造成多余网卡流量
网卡流量 服务器硬件
*磁盘IO
高并发 数据库连接数被占满max_connections默认100
高CPU使用率
CPU资源耗尽宕机
大表和大事务
记录行数超过千万行
表数据文件超10G
对DDL影响 5.5版本以前会锁表，5.5以后不会锁表但会引起主从延迟
修改表结构需要长时间锁表，造成长时间的主从延迟，影响正常数据的操作
处理数据库中的大表
分库分表把一张大表分成多个小表
分表主键的选择
分表后跨分区数据的查询和统计
历史数据归档(减少对前后端业务的影响)
归档那个时间点的选择
如何进行归档操作
事务
原子性 必须被视为一个不可分割的整体要么全部成功 要么全部失败
一致性 将数据库从一种一致性状态转换到另一种一致性状态，事务开始前和结束后数据完整性不被破坏
隔离性 一个事务在对数据库进行修改未提交完成时对于其他书屋是不可见的
持久性
大事务
定义 运行时间比较长，操作数据比较多的事务
锁定太多数据 造成大量阻塞和锁超时，回滚时间比较长，执行时间长，主从延迟
避免一次处理太多数据
从事务中移除不必要的SELECT操作
